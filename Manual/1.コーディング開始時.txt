このフローでは、**Pythonのバージョン管理**、**仮想環境の作成**、**Gitリポジトリの初期化**、そして**GitHubへのプッシュ**までを一気に行います。

-----

## 🚀 新しい開発プロジェクトの作成手順

この手順は、**個人PC**と**会社PC**のどちらから始めても構いません。ここでは、慣れている **Git Bashターミナル** を使用します。

### Step 1: プロジェクトフォルダの作成と移動

作業を始めるための新しいフォルダを作成し、その中へ移動します。

```bash
# プロジェクト名を "NewApp" と仮定
mkdir NewApp
cd NewApp
```

-----

### Step 2: Python バージョンの指定と仮想環境の構築

プロジェクト専用の独立した開発環境を作ります。

#### 1\. Python バージョンの指定

`pyenv` を使って、このプロジェクトで使うPythonのバージョンを指定します。（前回インストールした **3.12.5** を指定します）

```bash
pyenv local 3.12.5
```

#### 2\. 仮想環境の作成

プロジェクトフォルダ内に `.venv` という名前で仮想環境を作成します。

```bash
python -m venv .venv
```

#### 3\. 仮想環境の有効化

作成した環境を有効化し、今後インストールするライブラリがこのプロジェクト内だけに隔離されるようにします。

```bash
source .venv/Scripts/activate
```

  * **確認**: ターミナルのプロンプトの左端に **`(.venv)`** と表示されれば成功です。

-----

### Step 3: Gitリポジトリの初期化と必須設定

このフォルダをGitで管理できるようにし、無視すべきファイルを指定します。

#### 1\. Gitリポジトリの初期化

```bash
git init
```

#### 2\. `.gitignore` の作成

仮想環境ファイル (`.venv`) など、GitHubに上げてはいけないファイルを無視させる設定を作成します。

```bash
# Linux/macOSの場合:
# echo -e ".venv/\n__pycache__/\n*.pyc" > .gitignore

# Windowsで安全に作成する場合（VS Codeで開いて手動作成を推奨）:
# VS Codeを開きます
code .
```

  * VS Codeで `.gitignore` ファイルを作成し、少なくとも以下を記述して保存してください。

<!-- end list -->

```gitignore
# 仮想環境
.venv/
venv/

# Pythonバイトコード
__pycache__/
*.pyc
```

-----

### Step 4: コードの作成と依存関係の定義

実際にコードを書き始め、必要なライブラリをインストールします。

#### 1\. 動作確認用のコードを作成

VS Codeで `app.py` などのファイルを新規作成し、簡単なコードを記述して保存します。

#### 2\. ライブラリのインストール

プロジェクトに必要なライブラリをインストールします。（例：Webアプリなら `Flask`）

```bash
pip install Flask
```

#### 3\. 依存関係のリスト化 (最重要)

インストールしたライブラリを他のPC（またはチームメンバー）と共有するためにリスト化します。

```bash
pip freeze > requirements.txt
```

-----

### Step 5: 初回コミットとGitHubへのプッシュ

ローカルでの準備が整ったら、GitHubにリポジトリを作成し、コードをプッシュします。

#### 1\. GitHubでのリポジトリ作成

WebブラウザでGitHubにアクセスし、**新しい空のリポジトリ**（例: `new-app`）を作成します。**README、`.gitignore`、ライセンスは追加しないでください**（ローカルで作成したものを利用するため）。

#### 2\. ローカルでの初回コミット

作成したファイル（`app.py`, `.gitignore`, `requirements.txt`）をGitに登録し、コミットします。

```bash
git add .
git commit -m "feat: Initial project setup and environment definition"
```

#### 3\. リモートリポジトリの登録とプッシュ

GitHubで作成したリポジトリのURL（SSH形式：`git@github.com:ユーザー名/リポジトリ名.git`）をリモートとして登録し、プッシュします。

```bash
# リポジトリURLを置き換えてください
git remote add origin git@github.com:csd0910/new-app.git

# GitHubへの初回プッシュ
git push -u origin main
```

これで、GitHub上にあなたのプロジェクトが公開され、もう一方のPC（会社PCまたは個人PC）から **`git clone`** して開発をすぐに開始できるようになります。

-----

### 最終確認：もう一方のPCでの作業開始

もう一方のPCで作業を始めるときは、以下の3つのコマンドを実行するだけでOKです。

1.  **リポジトリのクローン**: `git clone git@github.com:csd0910/new-app.git`
2.  **フォルダ移動**: `cd new-app`
3.  **環境再構築**:
    ```bash
    python -m venv .venv
    source .venv/Scripts/activate
    pip install -r requirements.txt
    ```
